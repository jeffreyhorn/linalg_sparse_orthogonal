Why this is consistent with HCC actually firing
------------------------------------------------

A diagnostic run on a 5x5 grid confirms HCC + HEM produce
DIFFERENT matchings:

  fine n=25, ewgt=NULL
  HEM coarse n=13, HCC coarse n=14
    cmap[0]: HEM=0, HCC=4
    cmap[2]: HEM=9, HCC=0
    cmap[3]: HEM=5, HCC=9
    cmap[4]: HEM=8, HCC=9
    cmap[7]: HEM=9, HCC=6
  Total cmap differences: 12 / 25

So HCC's matching loop IS being called and IS producing different
cmap output.  The downstream ND pipeline (FM uncoarsening +
separator extraction + recursive ND on each side) converges to
the same final partition + nnz_L on every corpus fixture
regardless of the initial matching.

-------------------------------------

   secondary to primary headline-mover.  If HCC's matching change

   intermediate levels may have similar pass-through behavior --

3. **Days 6-8 (spectral bisection)** -- now THE headline-mover.
   Spectral bisection at the coarsest level is the only remaining
   axis that intervenes BEFORE FM at uncoarsening levels can
   wash it out; if spectral cuts the coarsest graph differently
   from GGGP's brute-force / greedy cuts, the better starting
   point should propagate up through the FM chain to a tighter
   finest-level cut.

4. **Day 9 (cross-corpus re-bench)** -- the combined-effect
   matrix is now expected to show HCC contributing ~0pp,
   intermediate-FM contributing TBD, spectral contributing the
   bulk of any 0.952x -> 0.85x movement.

5. **Day 10 (production-default decisions)** -- HCC default-flip
   threshold ("clear corpus-wide win on Pres_Poisson + small
   fixtures" per PLAN.md item 1) is unmet by Day 2's
   measurements.  Default is likely to STAY at heavy_edge unless
   spectral + intermediate FM expose HCC contributions in
   combination that don't appear when HCC runs alone.
